# AI开发经验分享

> 基于实际项目经验的AI开发工具和流程总结

## AI代码生成workflow

### 最佳实践流程

来自实际开发经验的总结：

1. **需求细化阶段**
   - 先用自然语言描述需求
   - 让AI帮助细化和完善需求
   - 改写成Markdown格式的文档

2. **开发文档生成**
   - 让AI根据细化后的文档生成开发文档
   - 然后进入实际开发阶段

3. **前端开发建议**
   - **除非是非常简单的demo，不要让AI从0写前端**
   - 找个基础的前端框架或者样例项目，生成就靠谱的多

## 推荐的AI开发工具组合

### 前端开发推荐工具

1. **v0.dev**
   - 用于前端界面设计
   - 每天有免费额度，小项目足够了
   - 尽量完成需求后再下载代码

2. **Cursor**
   - 用于后端API文档生成
   - 在前端项目中完成API集成
   - 细节微调和CSS优化

### 实际开发流程

```
v0.dev → 下载代码 → Cursor生成API文档 → 前端集成 → Cursor细节优化
```

而不是：
```
Cursor从0到1（不推荐）
```

## 代码审核workflow

### 推荐组合
- **cc max + codex**: 写代码 + 审核代码
- 配合起来还不错，单纯codex太慢

### UI设计workflow

来自实际项目经验：
1. **Google Stitch** 生成APP UI
2. 复制到 **Figma** 改成小程序样式
3. 让 **Cursor** 照着写
4. 确实有bug，改改就能搞定

Web开发就让Cursor直接写，交互bug肯定有，有时候要改好几遍。有些小问题自己能改肯定比AI改起来快一些。

## 经验总结

1. **不要完全依赖AI**: AI是辅助工具，不是万能解决方案
2. **循序渐进**: 从简单到复杂，逐步完善
3. **人工干预**: 关键步骤需要人工判断和调整
4. **工具组合**: 不同工具擅长不同领域，要合理搭配使用
